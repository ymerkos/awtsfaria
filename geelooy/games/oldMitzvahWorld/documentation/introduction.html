Game Maker API Documentation: Understanding the World of Awtsmoos Creation

Introduction

Welcome to the mystical and profound universe of Awtsmoos game development. This documentation is designed to help you understand the foundational elements and functionalities of the Awtsmoos game maker API. Even if you have no prior knowledge of game development or coding, this guide aims to provide a clear and vivid understanding of how to create and interact with various elements in the game world.

Core Concepts

  1. The World of Awtsmoos: At the heart of this API is the concept of creating a dynamic and interactive world. The 'world' in Awtsmoos is a space where players can explore, interact, and experience various events.

  2. Nivrayim (Creations): In Awtsmoos, everything within the game world – from static objects to interactive characters – is referred to as 'Nivrayim', meaning 'creations'. These creations can be anything from simple objects like stones to complex beings with their own behaviors.

  3. Placeholders and Custom Properties: One of the unique features of Awtsmoos is the ability to create placeholders in the world. These placeholders can be assigned custom properties, enabling developers to place new creations (Nivrayim) at specific positions within the game world.

Starting with the World

When you begin, you'll start with a base world – a foundational environment that players can interact with. This world is designed to be solid, meaning players can walk on it, collide with objects, and more.

Components of the Game World

  1. Resources: Resources are external assets that your game world will use. These can include soundtracks, 3D models (GLB files), textures, and more. In the given code, resources are defined as URLs pointing to various assets like soundtracks, portal models, city files, etc.

  2. Components: These are the building blocks of your game world. Components can be anything from audio files to 3D models. In Awtsmoos, you will see references to different components like soundTrack1, portalGLB, cityFile, etc.

  3. Assets: Assets refer to specific game elements within the world. These can be parts of a character, environmental elements, or any other items that make up your game world. For example, the goof asset in the code contains various parts like mouth, pupils, eyelids, etc.

Creating Nivrayim (Creations)

Creating Nivrayim involves defining their properties and behaviors. In the example code, various types of Nivrayim are defined:

  • Domem: Represents inanimate objects like the world itself.
  • Portal: Special objects that act as gateways or transition points.
  • Chai: Represents living beings.
  • Chossid: A specific type of character with detailed properties like height, speed, and interactive capabilities.
  • Medabeir: Characters capable of dialogue and interaction.

Each type of Nivra (singular for Nivrayim) has specific properties like path, placeholderName, proximity, etc., which determine how they behave and interact within the game world.

Interactivity and Behaviors

Nivrayim can have various interactive behaviors:

  • Events: These are actions or changes that occur in response to certain triggers. For instance, a character might respond to a keypress or an object might play a sound when collected.
  • Dialogue Trees: For interactive characters (like Medabeir), you can create complex dialogue trees with responses and actions.

Conclusion

The Awtsmoos game maker API offers a rich and flexible platform for creating immersive and interactive game worlds. By understanding the basic concepts of worlds, Nivrayim, components, and interactions, you can start building your own unique experiences in the realm of Awtsmoos. Dive in and let your creativity flow in this fascinating universe of endless possibilities!
