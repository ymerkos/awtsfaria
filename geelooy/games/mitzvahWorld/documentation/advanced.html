Advanced World Creation in Awtsmoos Game Maker API

Introduction

Creating a world in Awtsmoos is a multi-layered process that involves designing an environment, importing it into the game, and then defining specific behaviors and properties. This guide will delve deeper into the process, providing more examples and explaining the nuances of world creation.

Step-by-Step World Creation

  1. Designing the Low Poly World:

    • You can start by designing a low poly world. This could be done in any 3D modeling software like Blender.
    • Ensure the world is optimized for performance, especially if it's meant to be downloaded from various sources.

  2. Adding Animations:

    • In your 3D software, you can also create animations for different elements within your world.
    • These animations can be anything from moving platforms to animated characters.

  3. Exporting as GLB:

    • Once your world and its animations are ready, export the entire scene as a GLB file.
    • GLB format is preferred as it is a standard format for 3D scenes and models and is widely supported.

  4. Setting Placeholder Objects:

    • In your 3D software, add placeholder objects like cubes or empties in the positions where you want specific game elements (like a player or NPC) to appear.
    • Assign a custom property to these placeholders: "placeholder: [placeholderName]". For example, "placeholder: player" or "placeholder: npc_1".
    • This will ensure that when the GLB file is imported into Awtsmoos, these placeholders will automatically set the positions of the corresponding creations in the game code.

Properties and Behaviors

  1. Naming and Placeholder Matching:

    • In the Awtsmoos game code, each creation, like in the Domem object, should have a name and a placeholderName.
    • The placeholderName should match the value you set in the GLB file. If left blank, it defaults to the key of the object, like "world" in Domem: { world: { name: "world2", placeholderName: "world3" }}.

  2. Playing Animations - playChayoos :

    • To play an animation for a creation, use the playChayoos method.
    • Set the property playAll: true to play all animations associated with the creation.
    • For animations to be updated and played, ensure heesHawveh: true is set. This updates the creation and calls the animation methods.

  3. Making the World Solid - isSolid :

    • For a world to be solid and collidable, set isSolid: true.
    • This makes the entire world collidable and adds it to the static collision octree.
    • For complex meshes that might slow down the octree, use custom properties in the original GLB file.
    • Set notSolid: true on any parent object you want, and its children, to be non-collidable. This overrides the default solidity of the mesh.
    • Within a notSolid parent, if you set isAnywaysSolid: true on a child or sub-parent, that particular element (and its sub-children, if any) will be solid, despite an ancestor being notSolid.

Conclusion

Creating a world in Awtsmoos involves thoughtful design, careful planning of animations and placeholders, and a deep understanding of the properties and behaviors that define the game world. By following these steps and utilizing the unique features of the Awtsmoos API, you can create rich, interactive, and dynamic worlds that players can explore and interact with in meaningful ways.
